All issues have been fixed to ensure the project passes Phase 2 automated validation. Every score/reward value returned by any grader or endpoint is now strictly between 0.01 and 0.99 (never exactly 0.0 or 1.0).

### 1. server/graders/speedup_grader.py

### 3. server/graders/antipattern_grader.py

### 4. server/graders/index_grader.py
- Added `_clamp(v)` helper function: `return round(max(0.01, min(0.99, v)), 4)`
- Replaced all `return 0.001` with `return 0.01`
